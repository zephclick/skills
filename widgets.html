zui widget reference

← skills catalog

The pushed-UI JSON IR, one entry per kind (36 kinds, 35 streamable) — every field shown decodes on the device, every example is rendered through the device UI stack exactly as push_screen / notify would. Machine-readable: widgets.json.

Kinds marked not streamable carry their content in fields without wire keys yet — they parse but render empty; never push them standalone.

screen k=0 streamable

Screen — Full-screen root: a layout template + a column of children. The usual root of a push_screen tree.

fieldJSON keytypenotes
titletstringScreen title; rendered by the foundation/list/pinned layouts (suppressed when room chrome owns the title).
layoutlytuint0 = foundation (safe-area scrolling column, default), 1 = centered, 2 = list (sticky title + rows), 3 = pinned (scrolling content + fixed bottom CTA).
childrencarrayChild nodes, rendered top to bottom into the layout slot.

The pinned layout's fixed bottom CTA (zui.h screen.cta) has no CBOR key - lyt 3 over the wire renders without a bottom CTA.

{
  "k": "screen",
  "t": "Deploy",
  "c": [
    {
      "k": "text",
      "t": "Pick an environment on the next page"
    }
  ]
}

row k=7 streamable

Row — The standard list cell: leading icon, title (+ subtitle), trailing switch or chevron.

fieldJSON keytypenotes
titletstringCell title.
iconicstringLeading Lucide icon name (device icon subset).
subtitlesubstringSecondary line under the title.
toggletguint1 = trailing switch (whole-row tap flips it).
ononuintInitial switch state when tg = 1.
chevroncvuint1 = trailing chevron.
page refpgstringSwipe-up page ref ("app.export"): tap opens that page (v13 #358).
spinnerspuint1 = leading live arc spinner replaces the icon (working glyph, v13 #358).

The zui.h trailing value (val), icon color and custom icon image fields have no CBOR keys.

v13 #358: pg rows also render a trailing chevron; pre-v13 decoders skip pg/sp (inert row).

btn k=8 streamable

Button — Pill button: label + optional icon, variant/size/color styling, v12 role + physical-key binding.

fieldJSON keytypenotes
labeltstringOne line. Max 24 ASCII chars in modal bodies (18 when key-bound).
iconicstringLeading Lucide icon name.
variantvaruint0 = filled (default), 1 = tinted, 2 = gray, 3 = plain, 4 = success, 5 = destructive.
sizeszuint0 = md (default), 1 = sm, 2 = icon (circle), 3 = icon-sm.
colorcoluintPalette override (zui_color_ref): 0 = variant default, 1 = accent, 2 = ok, 3 = warn, 4 = danger, 5 = pixel, 6-8 = ink1-3, 9-10 = surface/surface2, 11-18 = vibrant blue/teal/green/amber/orange/crimson/iris/violet.
rolerolestringv12 dispatch: "submit" | "dismiss" (terminal), "event" (pings, stays), "next" | "prev" (local paging). JSON takes the name; the wire carries the uint.
keykeystringv12 physical-button binding: "up", "down", "up.double", "down.double". The button renders the key glyph; hold is never bindable.

An `id` makes taps report through the btn hook (modal action / widget event).

The zui.h hold (long-press fill) field has no CBOR key.

{
  "k": "screen",
  "c": [
    {
      "k": "text",
      "t": "Ready to deploy"
    },
    {
      "k": "btn",
      "id": "go",
      "t": "Deploy",
      "ic": "play",
      "role": "submit",
      "key": "up"
    }
  ]
}

Slider — Continuous 0-100 slider.

slot k=32 not streamable

Widget slot — Glance-face placeholder substituted by the slot resolver at render.

Firmware-only BY DESIGN (#328 decision): `idx`/`family` have no CBOR keys because substitution requires the device-side glance-face slot resolver (zui_render_set_slot_resolver) - outside glance composition a pushed slot can only render the dashed unassigned placeholder, so streaming one is never meaningful. image k=35 streamable

Image — A foreground image. The asset reference rides the node id; push the pixels over the asset plane first, then reference them.

The asset ref is the node `id`, resolved against the on-device asset store (#547). No image-specific CBOR keys decode today; push the encoded image over the XFER asset plane before a surface references it. A missing asset renders a placeholder, never a blank or crash.

  "k": "screen",
  "lyt": 1,
  "c": [
    {
      "k": "image",
      "id": "apple-music.art"
